# Capstan v0.24.4 Safety Gate Implementation Plan

> **For Hermes:** Execute with TDD and independent review. Do not start, stop, reinstall, or restart the live DS4 service during development.

**Goal:** Close the released safety and automation defects discovered in the 2026-07-09 truth pass before adding the Evidence Spine.

**Architecture:** Make manifest parsing strict at the boundary, centralize preflight enforcement for every heavyweight start path with an explicit same-port rotation exception, harden destructive cleanup through canonical path policy, make CLI exit status agree with structured results, and bound supervisor restart behavior. Keep all changes dependency-free and backward-compatible for valid v0.24 manifests.

**Tech Stack:** Python 3.11 stdlib, `unittest`, macOS LaunchAgents, GitHub Actions.

---

## Non-negotiable safety constraints

- No command may operate on the live DS4 service, PID, LaunchAgent, endpoint, or artifact directories during tests.
- All process tests use temporary manifests, ephemeral ports, and disposable child processes.
- All cleanup tests use temporary directories and files.
- Existing valid manifests remain compatible.
- Structured `ok=false`, failed readiness, or failed action results must produce non-zero CLI exit status.
- Rotation may tolerate only the current lane's occupied stable port when current and target share deployment identity; every other preflight check remains active.

## Task 1 — Strict manifest contract

**Files:**
- Modify: `modelctl/manifest.py`
- Modify/split: `tests/test_modelctl.py` or focused `tests/test_manifest_validation.py`

**RED tests:**

- reject `cleanup.safe = "false"`, `0`, and `1`;
- reject string booleans in `[health]` and `[fleet]`;
- reject numeric `model_id`, endpoint, prompt, and expected output;
- reject malformed endpoint URLs and invalid ports;
- reject negative/non-finite health, smoke, timeout, disk, and preflight limits;
- reject zero/negative startup timeout, smoke timeout, and max token counts;
- reject incompatible health sampling values;
- preserve all existing valid examples and live-manifest shape.

**Implementation:**

- add strict typed readers for strings, booleans, finite non-negative floats, positive ints, and URL/port validation;
- remove `str(...)`, `bool(...)`, and permissive numeric coercions at manifest boundaries;
- return contextual `ManifestError` messages;
- avoid broad unknown-key rejection in this cut unless tests prove compatibility; schema-version/unknown-key policy can follow in v0.25.

**Verification:** targeted validation tests, full 49-test baseline plus new tests, all example manifests parse.

## Task 2 — Truthful CLI exit semantics

**Files:**
- Modify: `modelctl/cli.py`
- Test: focused CLI tests

**RED tests:**

- `start --wait` returns non-zero when readiness fails;
- failed stop returns non-zero;
- failed recovery/rotate/promote/cleanup execution returns non-zero whenever payload says `ok=false`;
- status/list/show remain non-gating and return zero when they successfully report unhealthy state.

**Implementation:**

- route action results through one small `emit_result`/exit-code helper;
- preserve existing structured JSON shape;
- do not turn snapshot commands into gates.

## Task 3 — Centralized preflight before every start path

**Files:**
- Modify: `modelctl/runner.py`, `modelctl/fleet.py`, `modelctl/ops.py`, `modelctl/promote.py`
- Possibly create: `modelctl/safety.py` to avoid import cycles
- Test: focused lifecycle safety tests

**RED tests:**

- ordinary start is blocked by missing required path, occupied exclusive port, disk-floor failure, or swap ceiling;
- fleet recovery and daemon restart apply the same gate;
- rotate target and rollback apply preflight;
- same deployment identity tolerates only the current stable port during target planning/start;
- unrelated occupied ports remain blockers;
- a blocked preflight creates no process/PID side effect;
- rollback still works after a target readiness failure.

**Implementation:**

- centralize reusable preflight evaluation outside circular imports;
- make all heavyweight starts call the same gate;
- define a narrow `tolerated_ports` contract for same-role rotation only;
- include preflight evidence in structured results.

## Task 4 — Cleanup containment and active-use protection

**Files:**
- Modify: `modelctl/manifest.py`, `modelctl/ops.py`
- Test: focused cleanup safety tests

**RED tests:**

- reject `/`, home root, configured model/artifact roots, active cwd/binary/model paths, and ancestor deletion;
- reject path traversal and symlink escape;
- `safe=true` alone cannot bypass protected-path rules;
- `--force` cannot bypass hard-protected roots;
- missing paths remain harmless;
- safe disposable temp children delete successfully;
- failed deletion returns `ok=false` and non-zero CLI status.

**Implementation:**

- resolve canonical paths without following a deletion target after validation in a race-prone way;
- define hard protected roots and manifest-derived active paths;
- separate hard protection from force-overridable policy;
- return per-path blocked reasons.

## Task 5 — Bounded supervisor policy and durable events

**Files:**
- Modify: `modelctl/manifest.py`, `modelctl/ops.py`, `modelctl/service.py`, CLI parser
- Test: focused daemon/service tests

**RED tests:**

- warnings do not trigger restart;
- only declared critical failures can restart;
- consecutive-failure threshold, cooldown, and retry budget are enforced;
- restart events flush incrementally to an owner-only bounded NDJSON log;
- service policy is manifest-owned and `service diff` is authoritative without remembered flags;
- routine samples do not grow unbounded in memory or logs.

**Implementation:**

- keep defaults conservative and backward-compatible: restart remains explicit;
- add minimal manifest-owned supervisor policy;
- classify warning versus critical using existing health payload semantics;
- append compact events with rotation/size cap;
- do not build the full v0.25 receipt system here.

## Task 6 — Publication and repository gates

**Files:**
- Modify: `.github/workflows/release-artifacts.yml`
- Modify: `.github/workflows/publish-pypi.yml`
- Possibly modify: `.github/workflows/ci.yml`

**RED/static checks:**

- release and PyPI jobs run the same Python tests before publishing;
- release cannot upload if tests fail;
- avoid duplicate push+PR CI where practical without losing branch coverage.

**Implementation:**

- add explicit test/install/build gates to publication workflows or factor a reusable workflow if that stays smaller;
- keep GitHub release smoke-install checks.

## Task 7 — Docs, package, independent review

**Files:**
- Update `CHANGELOG.md`, `README.md`, and `docs/model-lifecycle.md`
- Bump version only after implementation is accepted

**Verification:**

1. `python3.11 -m unittest discover -s tests -v`
2. `python3.11 -m pytest -q` if pytest is available
3. `python3.11 -m build`
4. install wheel in an isolated venv; run `capstan --help`, `capstan version`, and safe temporary-manifest probes
5. `git diff --check`
6. scan changed lines for secrets, debug hooks, unsafe deletes, shell injection, and loose coercions
7. independent specification review
8. independent code-quality/security review
9. open PR; wait for all CI

## Two-hour checkpoint target

Minimum acceptable checkpoint:

- Tasks 1–4 complete with RED→GREEN evidence;
- full suite green;
- branch pushed and PR open;
- Tasks 5–6 either complete or clearly isolated with exact blockers.

Do not tag/release until all P0 defects, independent review findings, package verification, and CI are green.
